# teton-challenge

TPM-backed Wi-Fi provisioning for Linux IoT devices. A device running in SoftAP mode serves a TLS-protected credential form. The TLS certificate is signed by a per-device CA and the private key never leaves the TPM.

---

## Prerequisites

**Target hardware:** Raspberry Pi OS Bookworm or Ubuntu 22.04 LTS (x86-64), with a Wi-Fi interface.

**Required packages:**

```bash
sudo apt-get install -y \
  swtpm tpm2-tools tpm2-abrmd \
  libtss2-dev tpm2-openssl \
  openssl \
  python3 python3-pip \
  hostapd dnsmasq network-manager \
  libnss3-tools
```

**Python dependencies:**

```bash
pip install -r requirements.txt
```

---

## One-time setup

Initialises the software TPM, generates the device key inside it, and issues the device certificate signed by a local Teton demo CA:

```bash
sudo ./scripts/setup.sh
```

What it does:
- Wipes and restarts swtpm at `/tmp/tpm-state`
- Creates a primary key and a persistent RSA-2048 device key at handle `0x81000001`
- Generates `certs/teton-ca.crt` (demo CA) and `certs/device.crt` (device certificate)
- Verifies the cert chain with `openssl verify`

---

## Install CA certificate

The configurator browser must trust the Teton demo CA to avoid TLS warnings.

**Ubuntu / Debian:**

```bash
sudo ./scripts/install-ca.sh
```

This copies `certs/teton-ca.crt` into `/usr/local/share/ca-certificates/` and runs `update-ca-certificates`. It also imports the CA into every Firefox NSS profile found under `/home`.

**Other operating systems:** Import `certs/teton-ca.crt` manually into your browser's or OS certificate trust store.

## Run

```bash
sudo python3 device/provision.py
```

Expected terminal output (happy path):

```
2026-03-31 12:00:00 INFO State: INIT
2026-03-31 12:00:01 INFO State: INIT → AP_MODE
2026-03-31 12:00:01 INFO State: AP_MODE → PROVISIONED
2026-03-31 12:00:01 INFO State: PROVISIONED → CONNECTING
2026-03-31 12:00:05 INFO State: CONNECTING → ONLINE
2026-03-31 12:00:05 INFO Provisioning complete — device is connected.
```

**Provisioning steps:**

1. Device creates SoftAP `Teton-Device-0000` on `wlan0` (override with `PROVISION_IFACE=wlan1`)
2. Connect your browser machine to `Teton-Device-0000`
3. Navigate to `https://setup.teton-device.local` — padlock should appear with no warning
4. Enter the target Wi-Fi SSID and password, click **Provision**
5. Device disconnects from SoftAP, connects to the target network, and logs `ONLINE`

**Environment variables:**

| Variable | Default | Description |
|---|---|---|
| `PROVISION_IFACE` | `wlan0` | Wi-Fi interface for SoftAP |
| `PROVISION_TIMEOUT` | `600` | Seconds to wait for credential submission before error |

## Run tests

**Unit tests only** (no hardware required, always works):

```bash
pip install -r requirements-test.txt
python3 -m pytest tests/unit/ -v
```

**Full suite including integration tests** — requires `tpm2-openssl` and OpenSSL 3. On Ubuntu 22.04 / Raspberry Pi OS Bookworm these are available natively:

```bash
python3 -m pytest tests/ -v
```

**Full suite via Docker** — if `tpm2-openssl` is not installed on your machine (e.g. Debian 11, macOS), use the Docker runner instead. Requires Docker:

```bash
./scripts/docker-test.sh
```

This builds an Ubuntu 22.04 image on first run (cached afterwards) and executes the full suite inside it:

```
48 passed in ~30s
```

## Project layout

```
device/
  provision.py        State machine (INIT → AP_MODE → … → ONLINE)
  server.py           Flask HTTPS credential server
  wifi.py             SoftAP lifecycle (hostapd + dnsmasq) and nmcli connect
scripts/
  setup.sh            One-time TPM + PKI initialisation
  install-ca.sh       Install Teton CA into system trust store
certs/                Generated by setup.sh — not committed
tests/
  conftest.py         sys.path setup, shared fixtures (test CA, test device cert)
  unit/               Flask + state machine unit tests (no hardware required)
  integration/        Real swtpm TLS handshake tests (skipped if tpm2-openssl absent)
docs/
  architecture.md     System design, C4 diagrams, test strategy
```
